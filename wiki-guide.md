# Mobile App Installer 사용 가이드

APK/IPA 파일을 USB로 연결된 여러 모바일 디바이스에 **동시에 설치**할 수 있는 데스크톱 도구입니다.

## 사전 준비

### Android 디바이스 설치 시

#### 1. ADB (Android Debug Bridge) 설치

> **자동 설치 지원**: ADB가 설치되지 않은 PC에서 앱을 실행하면 상단에 "환경 설정" 배너가 표시됩니다. **`자동 설치`** 버튼을 클릭하면 Google 공식 platform-tools를 자동으로 다운로드하여 앱 폴더에 설치합니다. 시스템 PATH 수정 없이 바로 사용 가능합니다.

수동으로 설치하려면:

1. [Android SDK Platform-Tools](https://developer.android.com/tools/releases/platform-tools) 다운로드
2. 압축 해제 (예: `C:\platform-tools`)
3. **시스템 환경 변수 PATH에 추가**
   - `시작` → `환경 변수` 검색 → `시스템 환경 변수 편집`
   - `환경 변수` → `Path` → `편집` → `새로 만들기` → `C:\platform-tools` 입력
4. 터미널에서 확인:
   ```
   adb version
   ```

#### 2. 디바이스에서 USB 디버깅 활성화

1. `설정` → `휴대전화 정보` → `빌드 번호` **7번 터치** (개발자 옵션 활성화)
2. `설정` → `개발자 옵션` → **USB 디버깅** 켜기
3. USB 케이블로 PC에 연결
4. 디바이스 화면에 "USB 디버깅을 허용하시겠습니까?" 팝업이 뜨면 **허용** 선택

> **참고**: 팝업에서 "이 컴퓨터에서 항상 허용"을 체크하면 이후에 다시 묻지 않습니다.

### iOS 디바이스 설치 시

#### 1. iTunes 설치

- [iTunes 다운로드](https://www.apple.com/kr/itunes/) (Apple 공식 사이트 또는 Microsoft Store)
- iTunes 설치 후 **한 번 이상 실행**하여 Apple Mobile Device Driver가 설치되도록 합니다.

#### 2. 디바이스 신뢰 설정

1. iOS 디바이스를 USB로 PC에 연결
2. 디바이스 화면에 "이 컴퓨터를 신뢰하겠습니까?" 팝업이 뜨면 **신뢰** 선택
3. 디바이스 잠금 해제 상태 유지

## 실행 방법

1. `MobileAppInstaller.exe`를 더블클릭합니다.
2. 앱 창이 자동으로 열립니다.
3. 종료는 앱 창을 닫으면 됩니다 (우상단 X 버튼).

> **참고**: 첫 실행 시 Windows 방화벽 경고가 뜰 수 있습니다. 이 앱은 로컬(내 PC 안)에서만 동작하므로 "허용"해도 외부 통신은 발생하지 않습니다.

## 화면 구성

앱은 크게 3개 영역으로 구성됩니다.

### 연결된 디바이스 (좌상단)

- USB로 연결된 Android/iOS 디바이스 목록이 표시됩니다.
- 각 디바이스에는 다음 정보가 표시됩니다:
  - **디바이스 이름** (모델명)
  - **플랫폼 표시**: 초록 점 = Android, 파란 점 = iOS
  - **연결 상태**: 연결됨, 승인 필요 등
  - **저장 공간**: 여유 용량 / 전체 용량
- `새로고침` 버튼으로 디바이스 목록을 갱신할 수 있습니다.
- 체크박스로 설치 대상 디바이스를 선택합니다.

### 등록된 파일 (우상단)

- 업로드한 APK/IPA 파일 목록이 표시됩니다.
- 각 파일에는 다음 정보가 표시됩니다:
  - 파일 이름
  - 플랫폼 (Android/iOS)
  - 파일 크기
  - 패키지명 (APK) 또는 번들 ID (IPA)
- 파일은 앱을 종료 후 다시 실행해도 유지됩니다.
- 불필요한 파일은 삭제 버튼으로 제거할 수 있습니다.

### 설치 로그 (하단)

- 설치 과정의 실시간 로그가 표시됩니다.
- 성공/실패 여부와 에러 메시지가 한국어로 안내됩니다.

## 사용 방법

### 1. 파일 등록

- APK 또는 IPA 파일을 **파일 업로드 영역에 드래그앤드롭**하거나 클릭하여 선택합니다.
- 여러 파일을 등록할 수 있으며, 등록된 파일 목록에서 설치할 파일을 선택합니다.

### 2. 디바이스 연결 확인

- USB 케이블로 디바이스를 PC에 연결합니다.
- `새로고침` 버튼을 클릭하여 디바이스를 감지합니다.
- 디바이스 상태가 **"연결됨"**인지 확인합니다.

### 3. 설치 실행

| 방법 | 설명 |
|------|------|
| **선택 디바이스 설치** | 체크박스로 선택한 디바이스에만 설치 |
| **전체 디바이스 설치** | 연결된 모든 디바이스에 동시 설치 |

1. 드롭다운에서 **설치할 파일을 선택**합니다.
2. 필요 시 **"삭제 후 재설치"** 토글을 켭니다.
3. `선택 디바이스 설치` 또는 `전체 디바이스 설치` 버튼을 클릭합니다.
4. 하단 로그에서 설치 진행 상태를 확인합니다.

## 디바이스 상태 안내

| 상태 | 의미 | 조치 방법 |
|------|------|-----------|
| **연결됨** | 정상 연결, 설치 가능 | 없음 |

## 설치 옵션

### 덮어쓰기 설치 (기본)

- 기존에 설치된 앱 위에 새 버전을 덮어씁니다.
- **앱 데이터(로그인 정보, 설정 등)가 유지됩니다.**
- 서명이 동일한 경우에만 성공합니다.

### 삭제 후 재설치

- **"삭제 후 재설치" 토글**을 켜면 활성화됩니다.
- 기존 앱을 완전히 삭제한 뒤 새로 설치합니다.
- **앱 데이터가 초기화됩니다.**
- 서명이 달라도 설치 가능합니다.

> **사용 권장 상황**:
> - 서명이 변경된 빌드를 설치할 때
> - `INSTALL_FAILED_UPDATE_INCOMPATIBLE` 에러가 발생할 때
> - 앱 데이터를 초기 상태로 리셋하고 싶을 때

## 자주 발생하는 에러와 해결 방법

### Android

| 에러 | 원인 | 해결 방법 |
|------|------|-----------|
| **저장 공간 부족** | 디바이스 여유 공간 부족. APK 크기의 약 2~3배 공간 필요 (DEX 최적화 포함) | 불필요한 앱/파일 삭제하여 공간 확보 |
| **기존 설치와 서명이 다름** | 이전 빌드와 서명 키가 다른 APK | "삭제 후 재설치" 토글을 켜고 다시 설치 |
| **설치 버전이 기존보다 낮음** | 다운그레이드 시도 | "삭제 후 재설치" 토글을 켜고 다시 설치 |
| **디바이스 CPU 아키텍처와 호환되지 않는 APK** | APK가 해당 기기의 CPU ABI를 포함하지 않음 | 해당 기기를 지원하는 APK로 빌드 |
| **디바이스 Android 버전이 너무 낮습니다** | APK의 minSdkVersion이 디바이스보다 높음 | 더 높은 OS 버전의 디바이스 사용 |
| **설치 시간 초과** | 파일이 매우 크거나 USB 연결 불안정 | USB 케이블 교체 후 재시도 |

### iOS

| 에러 | 원인 | 해결 방법 |
|------|------|-----------|
| **tidevice가 설치되지 않았습니다** | iOS 지원 라이브러리 누락 | exe 배포판에 포함되어 있으므로 정상적으로는 발생하지 않음. 발생 시 재다운로드 |
| **설치 실패** | 프로비저닝 프로파일 문제 | IPA 파일의 서명/프로파일 확인 |

### 공통

| 증상 | 해결 방법 |
|------|-----------|
| 디바이스가 목록에 나타나지 않음 | USB 케이블 재연결 → 새로고침 버튼 클릭 |
| Android 디바이스가 "승인 필요"로 표시 | 디바이스 화면에서 USB 디버깅 허용 팝업 확인 |
| iOS 디바이스가 인식되지 않음 | iTunes가 설치되어 있는지 확인, 디바이스에서 "신뢰" 팝업 확인 |
| 앱 창이 뜨지 않음 | Windows에 Edge WebView2 런타임이 설치되어 있는지 확인 (Windows 10/11은 기본 포함) |

## FAQ

**Q. 인터넷 연결이 필요한가요?**
> 아니요. 이 앱은 완전히 오프라인으로 동작합니다. 모든 처리가 PC 로컬에서 이루어지며, 외부 서버에 접속하지 않습니다.

**Q. 다른 PC에서도 사용할 수 있나요?**
> 네. `MobileAppInstaller.exe` 파일 하나만 복사하면 됩니다. 단, 해당 PC에 **ADB** (Android용) 또는 **iTunes** (iOS용)가 설치되어 있어야 합니다.

**Q. 한 번에 몇 대까지 동시 설치가 가능한가요?**
> 소프트웨어 상 제한은 없습니다. USB 허브를 사용하여 여러 대를 동시에 연결할 수 있습니다. 실제로는 USB 대역폭과 PC 성능에 따라 결정됩니다.

**Q. APK/IPA 파일 크기 제한이 있나요?**
> 파일 크기 제한은 없습니다. 대용량 APK (500MB 이상) 설치 시 시간이 다소 소요될 수 있습니다. 디바이스에 APK 크기의 약 2~3배 여유 공간이 필요합니다.

**Q. 앱을 닫았다가 다시 열면 등록한 파일이 사라지나요?**
> 아니요. 등록한 파일은 exe 파일이 있는 폴더의 `uploads` 디렉토리에 저장되며, 앱을 재실행해도 유지됩니다.

**Q. Android와 iOS 디바이스를 동시에 연결해도 되나요?**
> 네. 동시에 연결 가능하며, 각 플랫폼에 맞는 파일(APK/IPA)을 선택하여 설치하면 됩니다. 단, APK는 Android에만, IPA는 iOS에만 설치됩니다.

**Q. "삭제 후 재설치"를 선택하면 앱 데이터도 삭제되나요?**
> 네. 기존 앱이 완전히 제거된 후 새로 설치되므로, 로그인 정보·캐시·설정 등 모든 앱 데이터가 초기화됩니다.
